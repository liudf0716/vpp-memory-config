基于 Chawrt 的 IPsec VPN 部署时
strongSwan 服务端注意事项

IKEv2 + XFRM + 地址池（Route-Based VPN）

01

架构选择：使用 swanctl 模式

  • 避免使用旧版 starter（ipsec.conf）
  • 推荐：swanctl + charon-systemd
dpkg -l | grep strongswan
02

删除旧版 strongswan-starter

systemctl status strongswan-starter apt remove strongswan-starter
03

安装 modern 架构组件

apt install strongswan-swanctl charon-systemd systemctl enable strongswan systemctl start strongswan systemctl status strongswan
04

创建 XFRM 虚拟接口

ip link add xfrm301 type xfrm dev eth0 if_id 301 ip link set xfrm301 up

未创建接口时： SA 建立成功但流量不通

05

地址池模式必须配置接口 IP

地址池示例：

addrs = 10.3.0.2-10.3.0.100

服务器必须拥有网关地址：

ip addr add 10.3.0.1/24 dev xfrm301
06

开启转发与路由

sysctl -w net.ipv4.ip_forward=1

按业务需求将对应网段路由到隧道接口（不要使用 default）：

ip route add 10.10.0.0/16 dev xfrm301 ip route add 172.16.20.0/24 dev xfrm301 ip route add 192.168.88.0/24 dev xfrm301
07

异地组网拓扑示意图

08

总结

  • 使用 swanctl + charon-systemd
  • 删除 strongswan-starter
  • 创建并配置 XFRM 接口
  • 地址池模式必须配置接口 IP
  • 按站点网段将静态路由指向 xfrm301（不要 default）
